# 阶段 1 验收清单（Web MVP）

## A. 功能验收

1. 规则创建

- 在页面新增 keyword 规则成功
- 在页面新增 topic 规则成功

2. 规则启停

- 点击规则卡片启停按钮后，规则状态变化并可刷新保持

3. 热点巡查

- 点击立即巡查后返回新事件数量
- 热点流出现来源、评分、判定理由

4. 趋势与通知

- 趋势面板展示按规则聚合结果
- 通知面板展示即时通知记录

5. 响应式

- 桌面端双栏布局可用
- 手机端单栏布局可用

## B. AI 验收

1. 无 key 模式

- /api/health 显示启发式降级
- 扫描流程仍可运行

2. 有 key 模式

- /api/health 显示 OpenRouter 已启用
- 热点判定理由可返回 AI 内容

## C. 接口验收

- GET /api/health 返回 ok=true
- POST /api/scan 返回 scannedAt 与 newEvents
- GET /api/hotspots 返回数组

## D. 持久化验收（SQLite + Prisma）

- 执行 `npx prisma migrate dev --name init` 成功
- 服务重启后，规则与历史热点数据可从数据库读取

## E. 测试验收

- npm test 全通过
